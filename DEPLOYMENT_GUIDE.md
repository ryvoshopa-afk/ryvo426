# دليل تشغيل ونشر منصة RYVO للإنتاج والنسخ الاحتياطي 🚀

مرحباً بك في الدليل الشامل المخصص لتشغيل، إدارة، ونشر منصة **RYVO** لبيع الدراجات وملحقاتها. تم تصميم النظام ببنية برمجية مرنة ومستقلة تتيح لك الإطلاق التجريبي فوراً دون سجل تجاري، مع قابلية توسع هائلة مستقبلاً.

---

## 📁 1. هيكلة النظام والتوافق البرمجي

تم عزل الطبقات البرمجية بالكامل لتسهيل التطوير المستقبلي:
* **قاعدة البيانات (Firestore Schema):** مرنة للغاية وتعتمد على الوثائق (Documents)، مما يتيح لك إضافة أي حقول إضافية لبوابات الدفع أو الموردين مستقبلاً دون الحاجة لتهيئة الجداول مجدداً.
* **إدارة المنتجات والطلبات:** تدعم حقول الموردين والتكلفة الفعلية والوزن والحجم بشكل مدمج استعداداً للربط الآلي أو للعمل بالنظام متعدد الموردين (Multi-Vendor).

---

## 🛠️ 2. تشغيل المشروع محلياً وفي بيئة التطوير

يستخدم المشروع تقنية **React + Vite** للواجهات الأمامية، و **Express (Node.js)** للمخدم الخلفي.

### متطلبات التشغيل الأساسية:
* مثبت حزم **Node.js** إصدار 18 أو أعلى.
* مدير الحزم **npm** (يأتي مدمجاً مع Node.js).

### خطوات التشغيل:
1. **تثبيت الحزم البرمجية:**
   ```bash
   npm install
   ```
2. **تشغيل خادم التطوير:**
   يقوم الخادم بتشغيل المخدم الخلفي والواجهة الأمامية بالتزامن:
   ```bash
   npm run dev
   ```
   *يعمل المشروع افتراضياً على الرابط: `http://localhost:3000`*

---

## 🔒 3. إعداد متغيرات البيئة (`.env`)

يجب إنشاء ملف باسم `.env` في المجلد الرئيسي للمشروع لربط الخدمات الحية وتأمين المفاتيح السرية.

```env
# منفذ تشغيل الخادم
PORT=3000

# وضع التشغيل (تطوير أو إنتاج)
NODE_ENV=production

# مفاتيح واجهة برمجة تطبيقات Firebase (العميل)
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id

# مفتاح محرك الذكاء الاصطناعي (Gemini) - مخفي تماماً في الخادم الخلفي
GEMINI_API_KEY=your_gemini_api_key_here

# بيانات الاتصال الخاصة بـ CJdropshipping API (للمستقبل)
CJ_API_KEY=your_cj_api_key_here
CJ_DEVELOPER_KEY=your_cj_developer_key_here
```

## 📦 4. بناء المشروع ونشره للإنتاج (Production Build & Deploy)

عند الرغبة في رفع الموقع على سيرفر حي (مثل **Cloud Run**، **VPS**، أو **Heroku**):

1. **بناء نسخة الإنتاج:**
   يقوم هذا الأمر بضغط الواجهات الأمامية وحزمها، وتجميع المخدم الخلفي في ملف واحد مدمج وسريع التشغيل داخل مجلد `dist/`:
   ```bash
   npm run build
   ```
2. **بدء تشغيل خادم الإنتاج السريع:**
   ```bash
   npm run start
   ```

### النشر على خوادم السحاب (على سبيل المثال Cloud Run أو Docker):
المشروع جاهز تماماً للحزم عبر `Docker`. يمكنك استخدام ملف `Dockerfile` يحتوي على:
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["npm", "run", "start"]
```

## 💾 5. استراتيجية النسخ الاحتياطي واستعادة البيانات (Database Backup)

بما أن قاعدة البيانات الأساسية هي **Google Cloud Firestore**، فلديك آليات نسخ احتياطي متطورة وتلقائية بالكامل تضمن حماية بيانات عملائك ومنتجاتك:

### الخيار أ: النسخ الاحتياطي التلقائي بضغطة زر (عبر Firebase Console)
1. توجه إلى منصة **Google Cloud Console** المرتبطة بمشروع Firebase الخاص بك.
2. ابحث عن خدمة **Firestore Backups**.
3. قم بتفعيل النسخ الاحتياطي اليومي وجدولته والاحتفاظ بنسخ لمدد تصل إلى 7 أو 30 يوماً تلقائياً دون كتابة أي كود.

### الخيار ب: التصدير اليدوي للبيانات (Manual Export via CLI)
يمكنك تصدير قاعدة البيانات بالكامل وحفظها في سحاب **Google Cloud Storage** عبر الأمر التالي باستخدام أداة `gcloud`:
```bash
gcloud firestore export gs://[اسم_مستودع_التخزين_الخاص_بك]
```
ولاستعادة البيانات في أي وقت لاحقاً:
```bash
gcloud firestore import gs://[اسم_مستودع_التخزين_الخاص_بك]/[مجلد_النسخة_الاحتياطية]
```

## ⚙️ 6. البنية التحتية الجاهزة للمستقبل (Future Integration)

1. **بوابات الدفع (Moyasar, Tap, Tamara, Tabby):**
   * تم توفير حقول مخصصة مثل `payment_method_raw` في الطلب.
   * لتفعيل أي بوابة مستقبلاً، تحتاج فقط إلى إضافتها كخيار في واجهة الدفع (`CheckoutModal`) وربط الـ Webhook الخاص بها مع خادم Express لاستقبال رسائل تأكيد الدفع وتحديث حالة الطلب إلى `processing` تلقائياً دون أي مساس بهيكل المنتجات أو هيكل قاعدة البيانات الحالي.
   
2. **الربط مع موردين خارجيين (CJdropshipping / AliExpress):**
   * البنية جاهزة ومثبتة بحقول مخصصة تدعم الاستيراد المباشر والمزامنة التلقائية.
   * عند توفر مفاتيح API الرسمية من CJ، يتوفر في الخادم الخلفي مسبقاً مسار `/api/dropshipping/import` ومسار معالجة الطلبات التلقائي لإرسال الطلب وحجز الشحن والحصول على رقم تتبع (`tracking_number`) ديناميكي حقيقي وتسجيله بالطلب.
